# Environment vs Code Drift Report

Compares one data product's source code snapshot against deployed BigQuery metadata and GCS artifacts to highlight drift between code and the selected GCP environment.

---

## Script

`generate_code_environment_drift_report.py`

---

## Outputs

Each run writes results to three destinations:

### 1. Local files (under `--output-dir`)

Timestamped folder, e.g. `outputs/my_product_20260514T120000Z/`:

| File | Contents |
|---|---|
| `report.md` | Human-readable discrepancy summary |
| `bigquery_findings.csv` | BigQuery drift: missing objects, schema drift, definition mismatches |
| `gcs_findings.csv` | GCS artifact drift |
| `inventory_snapshot.json` | Full structured snapshot of source and deployed inventories |

### 2. GCS bucket

All files are uploaded to `gs://<reports-gcs-bucket>/<reports-gcs-prefix>/<execution_id>/`.

### 3. BigQuery (`devops_reports` dataset)

| Table | What is written |
|---|---|
| `env_drift_findings` | One row per detected drift finding |
| `executions` | One row per repo processed (`triggered_by`, `duration_seconds`, `status`, `gcs_path`) |

---

## How To Run

### Inside Cloud Run (standard)

Triggered automatically by `entrypoint.sh` as `env_drift` report type. Runs once per repo found under `$SUBGROUP_ROOT`.

### Locally

Branch-based source comparison:

```bash
python generate_code_environment_drift_report.py \
  --project-name my_product \
  --repo-path /workspace/repos/fastossb/ndl_core/my_product \
  --gcp-project tefde-gcp-fastoss-prod \
  --reporting-project tefde-gcp-fastoss-dev \
  --git-ref production \
  --gcs-bucket fastoss-prod-composer-3 \
  --reports-gcs-bucket tefde-gcp-fastoss-dev-gcs-devops-reports
```

If `BQ_DATASET_NAMES` is not available in `.gitlab-ci.yml`, pass datasets explicitly:

```bash
python generate_code_environment_drift_report.py \
  --project-name my_product \
  --repo-path /workspace/repos/fastossb/ndl_core/my_product \
  --datasets dataset_one dataset_two
```

## Parameters

| Argument | Required | Default | Description |
|---|---|---|---|
| `--project-name` | yes | — | Data product name (used for output folder naming) |
| `--repo-path` | yes | — | Path to the local clone of the product repo |
| `--gcp-project` | no | — | GCP project for source GCS bucket (Composer) |
| `--reporting-project` | no | `--gcp-project` | GCP project for `devops_reports` BQ dataset |
| `--git-ref` | no | `production` | Git branch/ref for source comparison |
| `--gcs-bucket` | no | — | Source GCS bucket (Composer DAGs) |
| `--gcs-prefix` | no | — | Prefix inside the source GCS bucket |
| `--reports-gcs-bucket` | **yes** | — | GCS bucket for report output upload. **Required** — omitting it raises an error. |
| `--reports-gcs-prefix` | no | `env-drift` | Prefix inside the reports GCS bucket |
| `--output-dir` | no | `./outputs` | Local output root |
| `--datasets` | no | — | Override dataset list (bypasses `.gitlab-ci.yml` lookup) |
| `--source-mode` | no | `branch` | `branch` or `nexus` |

### Environment variables

| Variable | Description |
|---|---|
| `EXECUTION_ID` | Regenerated per repo by `entrypoint.sh` inside the `env_drift` loop — each repo gets its own UUID and its own `executions` row. |
| `TRIGGERED_BY` | Identity of the user who triggered the run; injected by the UI from the IAP header. Defaults to `"ui"`. |
| `NEXUS_USER` | Nexus username (only for `--source-mode nexus`) |
| `NEXUS_PASSWORD` | Nexus password (only for `--source-mode nexus`) |

---

## Prerequisites

- `git` available in PATH
- `gcloud` installed and authenticated (or running under Workload Identity in Cloud Run)
- BigQuery metadata access to the target project
- GCS read access to the Composer bucket
- `curl` available in PATH when using the default REST client

---

## Notes

- The script reads production-style settings from `.gitlab-ci.yml` when available.
- `--source-mode nexus` is supported for comparing released Nexus artifacts instead of a Git branch.
- `EXECUTION_ID` is required. It is auto-generated by `entrypoint.sh` when running inside Cloud Run.
